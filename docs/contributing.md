# Contributing

Contributions should be reproducible and reviewed. Use the repository `CONTRIBUTING.md` guidance and open a PR with an ADR if the change affects architecture.

Formatting notes:

- Use Markdown for content.
- Keep headings consistent.
- Add references and provenance for external sources.

Definition of done for documentation PRs

1. Content builds with `mkdocs build --strict`.
2. Claims are supported by runnable examples or references.
3. Operational guidance includes failure modes and mitigations.
